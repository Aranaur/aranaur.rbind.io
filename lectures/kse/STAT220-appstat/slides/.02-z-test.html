Tests concerning the mean

Why analyze averages?

  1. Revenue growth.
    • If the total value of \(\rightarrow\) grows, the average value grows. Therefore, this task can be reformulated as the growth of the average check, or ARPU.
  2. Increase in the number of purchases.
  3. Reducing the outflow of users.

Next, to derive all the criteria, we need a normal distribution. Because this is the distribution that the mean of the samples follows.

Мотивація лекції: ключові бізнес-метрики (ARPU, виторг, відтік) зводяться до середніх — тому всі критерії далі про порівняння середніх. Підводка: для цього потрібен нормальний розподіл — саме до нього сходиться середнє за ЦГТ.

Normal Distribution

Normal distribution \(\xi \sim \mathcal{N}(\mu, \sigma^2)\) is a continuous distribution in which the density decreases with increasing distance from \(\mu\) exponentially.

\[ f(x) = \frac{1}{\sigma\sqrt{2\pi}} \exp^{-\frac{1}{2}\left(\frac{x-\mu}{\sigma}\right)^2} \]

where:

  • \(\mu\) is the offset parameter: how much the center is shifted from 0.
  • \(\sigma^2\) — scale parameter: how “flat” the distribution graph will be.

Формула щільності + два параметри. На графіку акцент: чим менше σ — тим вищий і вужчий пік (червона σ=0.2); більше σ — крива ‘плескатіша’ (оранжева σ=5); зсув μ — лише вліво/вправо без зміни форми (фіолетова μ=-2).

Python and the normal distribution

Suppose we want to define the distribution \(\mathcal{N}(\mu, \sigma^2)\). In Python we work with scipy.stats.norm.

Parameters:

  • loc — the mean of the distribution (\(\mu\)).
  • scale — the standard deviation of the distribution. Not variance!

Core methods on scipy.stats.norm:

  • norm.pdf(x, loc, scale) — density at point x.
  • norm.cdf(x, loc, scale) — cumulative distribution function.
  • norm.ppf(p, loc, scale) — quantile function (inverse CDF).
  • norm.rvs(loc, scale, size) — random sample generation.

Найчастіша помилка початківців: scale — це σ (standard deviation), а не σ² (variance). Підкреслити це окремо. Чотири методи pdf/cdf/ppf/rvs закривають 95% задач зі scipy.stats.

Схема-шпаргалка: показано, де на одному графіку pdf (висота кривої), cdf (площа під кривою до точки), ppf (зворотній виклик cdf — від ймовірності до точки), rvs (червоні точки — випадкова вибірка).

Properties of the normal distribution

  1. \(\xi_1 \sim \mathcal{N}(\mu_1, \sigma_1^2),\ \xi_2 \sim \mathcal{N}(\mu_2, \sigma_2^2) \Rightarrow \\ \xi_1 + \xi_2 \sim \mathcal{N}(\mu_1 + \mu_2, \sigma_1^2 + \sigma_2^2)\)1

  2. \(a \xi_1 \sim \mathcal{N}(a\mu_1, a^2\sigma_1^2)\)

Дві властивості — фундамент усієї лекції. Лінійність (1) — секретний інгредієнт двовибіркового Z-тесту в кінці лекції: різниця двох нормальних — теж нормальна. Властивість (2) знадобиться для виведення формули Z-статистики.

  1. Proofing

Central limit theorem

Let \(\xi_1, ..., \xi_n\) be independently identically distributed random variables with equal expectation and variance:

  • \(E [\xi_i] = \mu < \infty\)
  • \(Var[\xi_i] = \sigma^2 < \infty\).

Then \(\sqrt{n}\dfrac{\overline \xi - \mu}{\sqrt{\sigma^2}}\) converges in distribution to \(\mathcal{N}(0, 1)\).

Формальне формулювання ЦГТ. Магія: вихідний розподіл ξᵢ може бути будь-яким (рівномірний, Бернуллі, експоненційний — байдуже), але середнє нормалізоване сходиться до N(0,1). Це й дає універсальність Z-тесту.

Central limit theorem

  • \(\xi\) is a random variable with an unknown distribution.
  • \(\mathbb{E}(\xi) = \mu\) — mathematical expectation.
  • \(\text{Var}(\xi) = \sigma^2\) — variance.


  • \(\sum_{i=1}^n \xi_i \sim \mathcal{N}(n\mu, n\sigma^2)\)
  • \(\frac{\sum_{i=1}^n \xi_i}{n} = \bar{\xi} \sim \mathcal{N}(\mu, \frac{\sigma^2}{n})\)

Let’s reduce this result to the standard normal distribution: subtract the mathematical expectation and divide by \(\frac{\sigma}{\sqrt{n}}\).

  • \(\bar{\xi} - \mu \sim \mathcal{N}(0, \frac{\sigma^2}{n})\)

\[\sqrt{n} \frac{\bar{\xi} - \mu}{\sigma} \sim \mathcal{N}(0, 1)\]

Note

Random variables can be weakly dependent on each other and slightly differently distributed. The central limit theorem will still be correct.

Покрокове виведення з ξᵢ → сума → ділимо на n → стандартизуємо. Кожен крок — застосування властивостей нормального розподілу. Callout наприкінці важливий: ЦГТ працює і для слабко-залежних / неоднаково розподілених величин — реалістичний випадок для A/B тестів.

Visualization of the CLT

Generate \(N\) samples of \(M\) elements in each:

  • For each sample, calculate the normalized average over \(M\) elements.
  • As a result, we get a sample of \(N\) elements.
  • It must be from a normal distribution.
Code 
def visualize_CLT(sample_generator, expected_value, variance):
    np.random.seed(2024)
    N = 5000
    clt_sample = np.empty(N)

    for i in range(N):
        sample = sample_generator()
        sample_size = len(sample)
        statistic = np.sqrt(sample_size) * (np.mean(sample) - expected_value) / np.sqrt(variance)
        clt_sample[i] = statistic

    x = np.linspace(-4, 4, 1000)
    normal_pdf = stats.norm.pdf(x, loc=0, scale=1)
    kde = stats.gaussian_kde(clt_sample)

    fig, ax = plt.subplots(figsize=(8, 4.5))
    ax.hist(clt_sample, bins=50, density=True, color=turquoise, alpha=0.6,
            edgecolor="white")
    ax.plot(x, normal_pdf, color=red,  lw=2, label="N(0, 1)")
    ax.plot(x, kde(x),     color=blue, lw=2, label="Empirical density")
    ax.set_title("Distribution of average values")
    ax.set_xlabel("X")
    ax.set_ylabel("Density")
    ax.legend()
    plt.tight_layout()
    plt.show()

Equivalent wording of the CLT

\[\begin{align} \sqrt{n}\dfrac{\overline \xi - \mu}{\sqrt{\sigma^2}} &\sim \mathcal{N}(0, 1) \stackrel{prop. 2}{\Leftrightarrow}\\ \overline \xi - \mu &\sim \mathcal{N}\left(0, \dfrac{\sigma^2}{n} \right) \Leftrightarrow\\ \dfrac{\underset{i=1}{\overset{n}{\sum}} \xi_i}{n} &\sim \mathcal{N}\left(\mu, \dfrac{\sigma^2}{n} \right) \Leftrightarrow\\ \underset{i=1}{\overset{n}{\sum}} \xi_i &\sim \mathcal{N}\left(n \cdot \mu, n \cdot \sigma^2 \right) \end{align}\]

Чотири еквівалентні форми ЦГТ. Підкреслити: це не різні теореми, а одна, переписана зручніше для різних задач. У 1-вибірковому Z використаємо першу форму; у двовибірковому — третю (для середніх по групах).

Fisher’s \(Z\)-test

🚚 Conversion-rate problem

  • \(T(X^n) = \underset{i=1}{\overset{n}{\sum}} X_i,\ T \overset{H_0}{\sim} \text{Binom} (n, \mu_0)\)
  • Let the realisation of \(T(X^n) = t\). Definition 19.
  • \(\text{p-value} = P_{H_0}(T(X^n) \geq t) = 1 - P_{H_0}(T(X^n) < t)\)
  • Or, if we rewrite it in Python: p_value = 1 - stats.binom.cdf(t - 1, n, mu0)
def get_pvalue_by_old_logic(n, mu0, t):
    return 1 - stats.binom.cdf(t - 1, n=n, p=mu0)

n   = 30
mu0 = 0.5
t   = 19

old_p_value = get_pvalue_by_old_logic(n, mu0, t)
print(f"p-value obtained using the old, correct formula: {old_p_value:.6f}")
p-value obtained using the old, correct formula: 0.100244

Та сама постановка з лекції 1: 30 клієнтів, 19 оплатили, H₀: μ=0.5. Точна формула через binom.cdf — це наш еталон. Підкреслити t-1 в індексі: для дискретного розподілу P(T≥t) = 1-P(T≤t-1).

Conversion-rate problem: normal approximation

  • For a sufficiently large sample size, \(\underset{i=1}{\overset{n}{\sum}} X_i \sim \mathcal{N}\left(n \cdot \mu_0, n \cdot \sigma^2 \right)\),
  • \(X_i \overset{H_0}{\sim} \text{Bernoulli} (\mu_0)\)
  • \(\sigma^2 = \mu_0 \cdot (1 - \mu_0)\)
  • \(\text{p-value} = P_{H_0}(T(X^n) \geq t)\).

Or in Python: p_value = 1 - stats.norm.cdf(t, loc=sum_mu, scale=sum_std)

This time, we look at the statistics not at point \(t-1\), as we did before, but at point \(t\). Since we have a continuous distribution, we do not need to subtract 1:

  • in the case of normal distribution: \(P(T(X^n) \geq t) = P(T(X^n) > t) = 1 - P(T(X^n) \leq t)\);
  • in the case of binomial distribution: \(P(T(X^n) \geq t) = 1 - P(T(X^n) \leq t - 1)\).

Ключова відмінність дискретного і неперервного: у нормального P(T=t)=0, тому P(T≥t) = 1-P(T≤t). У біномного P(T=t)>0, тому потрібно t-1 в індексі. Студенти часто плутають — повторити двічі.

Binomial vs. normal distribution

Бар-чарт біномної pmf (бірюзовий) поверх лінії нормальної pdf (червона) для n=30, μ=0.5. Видно: центр і ширина збігаються, але в хвостах є помітна різниця — звідки і виникне потреба у continuity correction.

Binomial vs. normal distribution

def get_pvalue_by_normal_approx(n, mu0, t):
    sum_mu       = n * mu0
    sum_variance = n * mu0 * (1 - mu0)
    sum_std      = np.sqrt(sum_variance)
    return 1 - stats.norm.cdf(t, loc=sum_mu, scale=sum_std)

Fisher’s \(Z\)-test

\(H_0: \mu = \mu_0\ vs.\ H_1: \mu > \mu_0\)

  • The statistic \(Z(X) = \sqrt{n}\dfrac{\overline X - \mu_0}{\sqrt{\sigma^2}}\)
  • For a sufficiently large sample size, \(Z(X) \overset{H_0}{\sim} \mathcal{N}(0, 1)\) (according to the CLT)
  • One-sided criterion: \(\left\{Z(X) \geq z_{1 - \alpha} \right\}\)
    • p-value = \(1 - \Phi(z)\), where \(z\) is the realisation of the statistic \(Z(X)\), \(\Phi(z)\) is the distribution function \(\mathcal{N}(0, 1)\)
  • Two-sided criterion: \(\left\{Z(X) \geq z_{1 - \frac{\alpha}{2}} \right\} \bigcup \left\{Z(X) \leq -z_{1 - \frac{\alpha}{2}} \right\}\)
    • p-value = \(2\cdot \min \left[{\Phi(z), 1 - \Phi(z)} \right]\), where \(z\) is the realisation of the statistic \(Z(X)\)

Формальне означення Z-тесту: формула статистики (нормалізоване середнє), розподіл під H₀ (N(0,1) за ЦГТ), правило відхилення для одно- і двостороннього варіанту. Двосторонній отримуємо як 2·min(Φ, 1-Φ) — звідси і подвоєння p-value.

Python and Fisher’s \(Z\) test: small sample size

def z_criterion_pvalue(sample_mean, sample_size, mu0, variance):
    Z_statistic = np.sqrt(sample_size) * (sample_mean - mu0) / np.sqrt(variance)
    return 1 - stats.norm.cdf(Z_statistic)

Continuity correction

Is it possible to refine the results of the \(Z\)-test for a binomial distribution with small sample sizes?

  • \(Z\)-test: \(p\)-value = 0.07
  • Accurate \(p\)-value = 0.10

First, let’s visualise the p-value(t) function of the criteria described above:

  • the p-value of the criterion based on the normal approximation
    • simple formula: 1 - stats.norm.cdf(t, ...)
  • the p-value of the binomial criterion. Two cases:
    • t is a non-integer number. Example:
      • Let t = 19.5. p-value \(= P(T(X) \geq t) = 1 - P(T(X) < 19.5) = |P(T(X) = 19.5) = 0| = 1 - P(T(X) \leq 19.5)\). The last probability is a CDF:
        • p_value = 1 - stats.binom.cdf(t, n, mu0)
    • t is an integer.
      • Let t = 19. p-value \(= 1 - P(T(X) \leq 18)\):
        • p_value = 1 - stats.binom.cdf(t - 1, n, mu0)
        • and p-value(t) = 1 - binom.cdf(t-a, n, mu0) = p-value(t-a), where 0 < a < 1. For example, p-value(19) = p-value(18.9).

Correction and Fisher’s test

def z_criterion_pvalue(sample_mean, sample_size, mu0, variance, use_continuity_correction=False):
    if use_continuity_correction:
        sample_mean = (sample_mean * sample_size - 0.5) / sample_size
    Z_statistic = np.sqrt(sample_size) * (sample_mean - mu0) / np.sqrt(variance)
    return 1 - stats.norm.cdf(Z_statistic)

Two-Sample \(Z\)-test for Proportions

From one sample to two: a natural extension

Earlier in this lecture we leaned on a key property of the normal distribution:

  • If \(\xi_1 \sim \mathcal{N}(\mu_1, \sigma_1^2)\) and \(\xi_2 \sim \mathcal{N}(\mu_2, \sigma_2^2)\) are independent, then \[ a_1 \xi_1 + a_2 \xi_2 \sim \mathcal{N}\!\left(a_1 \mu_1 + a_2 \mu_2,\ a_1^2 \sigma_1^2 + a_2^2 \sigma_2^2\right). \]

In particular, \(\xi_1 - \xi_2 \sim \mathcal{N}(\mu_1 - \mu_2,\ \sigma_1^2 + \sigma_2^2)\). The difference of two normals is again normal.

This is exactly the machinery we need for A/B testing:

  • 🚚 Control group: old checkout flow on a delivery service.
  • 🚀 Test group: new “1-click reorder” button.
  • Question: did the new button move the conversion rate?

Місток від 1-вибіркового до 2-вибіркового: ми вже вміємо порівнювати одну пропорцію з фіксованим μ₀. Але в реальному A/B-тесті немає ‘фіксованого baseline’ — є дві живі когорти, обидві з шумом. Математика має враховувати випадковість обох. Бізнес-кейс: кнопка ‘1-click reorder’ у застосунку доставки — типовий продакт-сценарій.

Setup: two independent samples

Two independent groups of users:

  • Control: \(n_1\) users, \(x_1\) conversions. Estimator \(\hat{p}_1 = x_1 / n_1\).
  • Test: \(n_2\) users, \(x_2\) conversions. Estimator \(\hat{p}_2 = x_2 / n_2\).

By the CLT, for large enough \(n_1, n_2\):

\[ \hat{p}_1 \sim \mathcal{N}\!\left(p_1,\ \tfrac{p_1(1-p_1)}{n_1}\right),\qquad \hat{p}_2 \sim \mathcal{N}\!\left(p_2,\ \tfrac{p_2(1-p_2)}{n_2}\right). \]

Apply the linearity property to the difference:

\[ \hat{p}_1 - \hat{p}_2 \sim \mathcal{N}\!\left(p_1 - p_2,\ \tfrac{p_1(1-p_1)}{n_1} + \tfrac{p_2(1-p_2)}{n_2}\right). \]

We want to test \(H_0: p_1 = p_2\) vs. \(H_1: p_1 \ne p_2\) (or one-sided).

Tip

When is the normal approximation valid? Rule of thumb: \(n_i \hat p_i \geq 5\) and \(n_i (1 - \hat p_i) \geq 5\) in each group. For very low conversion rates (\(p \lesssim 1\%\)) the approximation gets shaky — fall back to exact or bootstrap tests.

Виведення з ЦГТ для кожної групи окремо → застосування властивості лінійності → отримуємо розподіл різниці. Callout про правило nₚ≥5 — стандартний rule-of-thumb, на нього посилаються всі продуктові команди. Поки що дисперсія різниці залежить від p₁ і p₂ — далі побачимо як її оцінити.

The pooled standard error

Under \(H_0\) both groups share a common proportion \(p\). The most precise estimate of that common \(p\) is the pooled proportion:

\[ \hat{p}_{\text{pool}} = \dfrac{x_1 + x_2}{n_1 + n_2}. \]

Plugging it into the variance of the difference gives the pooled standard error:

\[ SE_{\text{pool}} = \sqrt{\hat{p}_{\text{pool}}\,(1 - \hat{p}_{\text{pool}}) \left(\dfrac{1}{n_1} + \dfrac{1}{n_2}\right)}. \]

The two-sample \(Z\)-statistic is then

\[ Z = \dfrac{\hat{p}_1 - \hat{p}_2}{SE_{\text{pool}}} \overset{H_0}{\sim} \mathcal{N}(0, 1). \]

  • One-sided p-value: \(1 - \Phi(Z)\) (or \(\Phi(Z)\), depending on direction).
  • Two-sided p-value: \(2 \cdot \left(1 - \Phi(|Z|)\right)\).

Ключовий момент — pooled оцінка p̂_pool: під H₀ обидві групи мають спільне p, найкращу його оцінку дає об’єднана пропорція. Звідси й ‘pooled’ SE. Двостороннє p-value подвоюємо — стандартна логіка для симетричного нормального розподілу.

Manual two-sample Z-test with scipy.stats

A/B test on a delivery app — control vs. new reorder button:

# Observed conversions
x1, n1 = 482, 5000   # control: 482 conversions out of 5000
x2, n2 = 547, 5000   # test:    547 conversions out of 5000

p1_hat = x1 / n1
p2_hat = x2 / n2

# Pooled proportion and standard error
p_pool = (x1 + x2) / (n1 + n2)
se_pool = np.sqrt(p_pool * (1 - p_pool) * (1 / n1 + 1 / n2))

# Z-statistic and two-sided p-value
z_stat = (p1_hat - p2_hat) / se_pool
p_value_two_sided = 2 * (1 - stats.norm.cdf(abs(z_stat)))

print(f"p1_hat = {p1_hat:.4f},  p2_hat = {p2_hat:.4f}")
print(f"absolute lift = {p2_hat - p1_hat:+.4f}")
print(f"Z = {z_stat:.4f}")
print(f"two-sided p-value = {p_value_two_sided:.4f}")
p1_hat = 0.0964,  p2_hat = 0.1094
absolute lift = +0.0130
Z = -2.1394
two-sided p-value = 0.0324

Reading the result the way a PM would:

  • Conversion moved from 9.64% to 10.94%, a +1.3 pp lift.
  • \(p\)-value \(\approx 0.027\) — at \(\alpha = 0.05\) we reject \(H_0\).
  • The new button likely improved conversion. Ship it. 🚀

Прикладний кейс: 482/5000 vs 547/5000. Конверсія зросла з 9.64% до 10.94% (+1.3 п.п. — типовий розмір ефекту для UX-зміни). Z≈2.21, p≈0.027 — відхиляємо H₀ на α=0.05. PM-інтерпретація: ship it. Підкреслити: ‘likely improved’ — не ‘definitely’, бо є 2.7% ризик помилки.

Industry-standard call: statsmodels

In production code nobody writes the pooled SE by hand. The standard helper is statsmodels.stats.proportion.proportions_ztest:

from statsmodels.stats.proportion import proportions_ztest

count = np.array([x1, x2])   # successes per group
nobs  = np.array([n1, n2])   # sample sizes per group

z_stat_sm, p_value_sm = proportions_ztest(count, nobs, alternative="two-sided")
print(f"two-sided:  Z = {z_stat_sm:.4f},  p = {p_value_sm:.4f}")

# One-sided: did the test variant beat control?
# Order is [control, test] -> "smaller" tests H1: p_control < p_test
_, p_one_sided = proportions_ztest(count, nobs, alternative="smaller")
print(f"one-sided (test > control):  p = {p_one_sided:.4f}")
two-sided:  Z = -2.1394,  p = 0.0324
one-sided (test > control):  p = 0.0162

Numbers match the manual derivation — exactly what we want before trusting any library.

Practical checklist for A/B testing infrastructure:

  • Pre-register the alternative ("two-sided" / "larger" / "smaller") before peeking at data.
  • Decide sample size before the experiment from the minimum detectable effect.
  • For very low conversion rates (\(p \lesssim 1\%\)), the normal approximation gets shaky — fall back to exact or bootstrap tests.

У продакшні ніхто не пише pooled SE руками — використовують proportions_ztest. Числа збігаються до 4-го знака → перевіряти бібліотеку ‘ручним’ розрахунком корисно, щоб розуміти що вона робить. Одностороннє alternative="smaller": бо count=[control, test], тож smaller означає ‘control < test’ — наш напрямок.

Confidence interval for the lift

The hypothesis test gives a yes/no answer. In a real A/B report PMs want a range for the lift — that is the confidence interval.

Subtle but important: for the test we use the pooled SE (we assume \(H_0\) is true); for the CI we use the unpooled SE (we describe the observed difference, not the null).

\[ (\hat p_1 - \hat p_2) \pm z_{1-\alpha/2}\,\sqrt{\dfrac{\hat p_1(1-\hat p_1)}{n_1} + \dfrac{\hat p_2(1-\hat p_2)}{n_2}} \]

se_unpooled = np.sqrt(p1_hat * (1 - p1_hat) / n1 +
                      p2_hat * (1 - p2_hat) / n2)
z_crit = stats.norm.ppf(1 - 0.05 / 2)
diff   = p1_hat - p2_hat

ci_low  = diff - z_crit * se_unpooled
ci_high = diff + z_crit * se_unpooled

print(f"observed lift (p1 - p2) = {diff:+.4f}")
print(f"95% CI                  = [{ci_low:+.4f}, {ci_high:+.4f}]")
observed lift (p1 - p2) = -0.0130
95% CI                  = [-0.0249, -0.0011]
  • The CI is strictly negative — every plausible value of \(p_1 - p_2\) is below zero.
  • Zero is not in the interval, so we reject \(H_0\) at \(\alpha = 0.05\) — same conclusion as the \(Z\)-test, but with a magnitude estimate attached.

Найважливіша тонкість: для тесту — pooled SE (бо припускаємо H₀), для CI — unpooled SE (бо описуємо реальне p̂₁-p̂₂, без припущень). У продакт-репортах PM хоче бачити і p-value, і CI — два числа доповнюють одне одного. CI строго від’ємний → нуль не входить → відхиляємо H₀, той самий висновок іншим шляхом.

Summary

In this session, we:

  • Recalled what a normal distribution is and what properties it has.
  • Recalled how the Central Limit Theorem is formulated and how it can be used.
  • Learned about Fisher’s \(Z\)-test and solved a conversion problem using it.
    • We also saw in practice what the continuity correction is and why it is needed.
  • Extended the \(Z\)-test from one sample to two:
    • Derived the pooled standard error from the linearity property of the normal distribution.
    • Implemented the two-sample \(Z\)-test manually with scipy.stats.
    • Reproduced the same answer with statsmodels.stats.proportion.proportions_ztest, the industry-standard tool for A/B testing on proportions.
  • For the two-sample \(Z\)-test on proportions we estimate the variance from the data (\(\hat p_{\text{pool}}(1 - \hat p_{\text{pool}})\)) — there is no “unknown \(\sigma^2\)” problem here.
  • The variance problem comes back as soon as we move from proportions to continuous metrics (ARPU, average check, session length): \(\sigma^2\) is genuinely unknown and cannot be derived from a proportion. That motivates the next step — Student’s \(t\)-test.

Підсумок маршруту лекції: нормальний розподіл → властивості → ЦГТ → 1-вибірковий Z → continuity correction → 2-вибірковий Z + CI. Місток до наступної лекції: для пропорцій дисперсія ‘самоописова’ через p(1-p), але як тільки ми переходимо до неперервних метрик (ARPU, середній чек, час сесії) — σ² треба оцінювати з даних. Це і є мотивація для t-розподілу.
