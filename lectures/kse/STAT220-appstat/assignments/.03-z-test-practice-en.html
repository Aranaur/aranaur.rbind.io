What we are doing today

In the lecture we built the Z-test piece by piece: the normal distribution, the CLT, the Z-statistic, continuity correction, and finally the two-sample Z-test for proportions with a confidence interval. Today we assemble all of it into one tool — first our own my_z_test(), then its two-sample sibling my_two_sample_z_test(), similar in spirit to statsmodels.stats.proportion.proportions_ztest.

NoteRules of the game
  • We do not use the ready-made proportions_ztest — we write everything ourselves via scipy.stats.norm (norm.cdf, norm.ppf).
  • Each version is a small extension of the previous one.
  • Every final step (Final, full cycle) reuses previous functions instead of duplicating them.
  • After every version there is a “Your turn” block with a small exercise plus a collapsible Solution.

1 The case: GreenBowl food delivery app

We work on GreenBowl — a healthy-food delivery app. The product team added a “1-click reorder” button on the home screen: a single tap re-creates the previous order. Before rolling the feature out to everyone, we want to confirm that it actually lifts the repeat-order metric.

  • Unit of observation: one user after the first order.
  • Success metric: the user placed another order within 7 days (1) or not (0).
  • Model: \(X_i \sim \text{Bernoulli}(\mu)\), success count \(T = \sum X_i \sim \text{Binomial}(n, \mu)\).
  • Baseline: \(\mu_0 = 0.10\) (3 months of historical data).
  • Business MDE: an effect smaller than +1 pp is not worth shipping.
  • Direction of interest: only upward → one-sided test.
  • Significance level: \(\alpha = 0.05\).
NoteWhy a Z-test, not an exact binomial test?

GreenBowl has hundreds of thousands of users. At those sample sizes the normal approximation is very accurate, and the Z-test is orders of magnitude faster than the exact binomial on \(n > 10^4\). That speed is the reason the Z-test exists at all.

2 Version 1 — Z-statistic and p-value

The simplest version: given \(n\), \(\mu_0\), and the observed count \(q\), return the one-sided \(p\)-value for \(H_1: \mu > \mu_0\).

\[ Z = \sqrt{n}\,\dfrac{\hat{p} - \mu_0}{\sqrt{\mu_0(1-\mu_0)}},\qquad p\text{-value} = 1 - \Phi(Z) \]

def my_z_test(q, n, mu0=0.5):
    """
    Version 1. One-sided Z-test for a proportion, H1: mu > mu0.
    """
    p_hat = q / n
    se    = np.sqrt(mu0 * (1 - mu0) / n)
    z     = (p_hat - mu0) / se
    return 1 - norm.cdf(z)

Let’s try it on our case. Marketing brought 2 000 new users into the test cohort, and 226 of them placed a repeat order within 7 days.

\(p < 0.05\) — we have evidence in favour of \(H_1\). The observed 11.3% conversion is statistically higher than the historical 10%. But do not ship yet — the effect may sit on the edge of business significance. We will add a CI next to see the size.

Interpretation. The critical lift scales as \(1/\sqrt{n}\) because \(SE \propto 1/\sqrt{n}\). So a 4-fold increase in \(n\) cuts the critical lift in half, not by a factor of 4. This is the fundamental A/B trade-off: detecting an effect half the size takes four times the users.

3 Version 2 — three types of alternative

The anti-fraud team sometimes wants to check whether the metric dropped (one-sided less); the product team occasionally wants “did anything change at all” (two-sided). We add an alternative parameter.

def my_z_test(q, n, mu0=0.5, alternative="greater"):
    """
    Version 2. Adds an `alternative` parameter.

    alternative : {"greater", "less", "two-sided"}
    """
    p_hat = q / n
    se    = np.sqrt(mu0 * (1 - mu0) / n)
    z     = (p_hat - mu0) / se

    if alternative == "greater":
        return 1 - norm.cdf(z)
    if alternative == "less":
        return norm.cdf(z)
    if alternative == "two-sided":
        return 2 * (1 - norm.cdf(abs(z)))

    raise ValueError(f"Unknown alternative: {alternative!r}")
for alt in ("greater", "less", "two-sided"):
    p = my_z_test(q=226, n=2000, mu0=0.10, alternative=alt)
    print(f"alternative={alt:>10s}  →  p-value = {p:.4f}")
alternative=   greater  →  p-value = 0.0263
alternative=      less  →  p-value = 0.9737
alternative= two-sided  →  p-value = 0.0526

For the same data the one-sided \(p\)-value is exactly half of the two-sided one — because the normal distribution is symmetric. This is not a coincidence, it follows from \(2 \cdot (1 - \Phi(|z|))\).

WarningWhen two-sided is mandatory

If a product has a real risk of causing harm (e.g. a new paywall that could either lift or sink conversion) — use two-sided. In the GreenBowl case the risk is one-sided: a worse button we simply do not ship.

General formula: \(p_{\text{two-sided}} = 2 \cdot \min(p_{\text{greater}},\ p_{\text{less}})\). We always take the tail where the observation is “more extreme”. When \(\hat p > \mu_0\) (\(z>0\)) — the right tail is more extreme, so \(p_{\text{two}} = 2 \cdot p_{\text{greater}}\). When \(\hat p < \mu_0\) (\(z<0\)) — the left tail, so \(p_{\text{two}} = 2 \cdot p_{\text{less}}\).

Why is it strict for Z but only approximate for the binomial? Because the normal distribution is symmetric around zero: \(P(Z > z) = P(Z < -z)\). The binomial with \(\mu_0 \ne 0.5\) is asymmetric, so the two tail probabilities are not equal, and doubling one tail gives only an approximate \(p_{\text{two}}\).

4 Version 3 — continuity correction

For small \(n\) the Z-test systematically underestimates the \(p\)-value relative to the exact binomial. In the lecture we patched this by shifting the normal by \(-0.5\). We add a use_cc flag.

def my_z_test(q, n, mu0=0.5, alternative="greater", use_cc=False):
    """
    Version 3. Adds continuity correction (recommended for n < ~100).
    """
    p_hat = q / n
    if use_cc:
        # Shift the observed value by 0.5/n towards mu0 — that is the correction
        if alternative == "greater":
            p_hat = (q - 0.5) / n
        elif alternative == "less":
            p_hat = (q + 0.5) / n
        else:
            # two-sided: shift towards mu0
            p_hat = (q - 0.5) / n if p_hat >= mu0 else (q + 0.5) / n

    se = np.sqrt(mu0 * (1 - mu0) / n)
    z  = (p_hat - mu0) / se

    if alternative == "greater":
        return 1 - norm.cdf(z)
    if alternative == "less":
        return norm.cdf(z)
    if alternative == "two-sided":
        return 2 * (1 - norm.cdf(abs(z)))

    raise ValueError(f"Unknown alternative: {alternative!r}")

  • Exact binomial: \(p \approx 0.058\) → do not reject \(H_0\). Not enough evidence.

On small \(n\) the normal approximation systematically underestimates the \(p\)-value in the tails. Without the correction you will “find” effects that are not there — and ship features on the back of noise. Continuity correction puts the decision back into the safe zone.

5 Version 4 — confidence interval for \(\mu\)

Instead of yes/no the business wants a range for the true conversion. The simplest CI is the Wald one:

\[ \hat{p} \pm z_{1-\alpha/2}\,\sqrt{\dfrac{\hat{p}(1-\hat{p})}{n}} \]

def my_z_test(q, n, mu0=0.5, alternative="greater",
              use_cc=False, alpha=0.05, conf_int=False):
    """
    Version 4. Adds conf_int=True → returns a dict with p-value and CI.
    The CI type matches the alternative:
      two-sided → [p̂ - z₁₋α/₂·SE, p̂ + z₁₋α/₂·SE]
      greater   → one-sided lower bound: [p̂ - z₁₋α·SE, 1]
      less      → one-sided upper bound: [0, p̂ + z₁₋α·SE]
    """
    p_hat_raw = q / n

    p_hat = p_hat_raw
    if use_cc:
        if alternative == "greater":
            p_hat = (q - 0.5) / n
        elif alternative == "less":
            p_hat = (q + 0.5) / n
        else:
            p_hat = (q - 0.5) / n if p_hat_raw >= mu0 else (q + 0.5) / n

    se_null = np.sqrt(mu0 * (1 - mu0) / n)
    z       = (p_hat - mu0) / se_null

    if alternative == "greater":
        p_value = 1 - norm.cdf(z)
    elif alternative == "less":
        p_value = norm.cdf(z)
    elif alternative == "two-sided":
        p_value = 2 * (1 - norm.cdf(abs(z)))
    else:
        raise ValueError(f"Unknown alternative: {alternative!r}")

    if not conf_int:
        return p_value

    # Wald CI uses SE based on p_hat_raw (not mu0!).
    se_emp = np.sqrt(p_hat_raw * (1 - p_hat_raw) / n)

    if alternative == "two-sided":
        z_crit  = norm.ppf(1 - alpha / 2)
        ci_low  = p_hat_raw - z_crit * se_emp
        ci_high = p_hat_raw + z_crit * se_emp
    elif alternative == "greater":
        z_crit  = norm.ppf(1 - alpha)
        ci_low  = p_hat_raw - z_crit * se_emp
        ci_high = 1.0
    else:  # less
        z_crit  = norm.ppf(1 - alpha)
        ci_low  = 0.0
        ci_high = p_hat_raw + z_crit * se_emp

    return {"p_value": p_value, "ci_low": ci_low, "ci_high": ci_high}
ImportantSubtle but important: SE for the test vs SE for the CI

For the test use \(\sqrt{\mu_0(1-\mu_0)/n}\) (we assume \(H_0\)). For the CI use \(\sqrt{\hat{p}(1-\hat{p})/n}\) (we describe what we observed, no assumption). The same pooled-vs-unpooled idea will reappear in the two-sample test.

NoteWhen Wald CI breaks

The Wald interval can fall outside \([0, 1]\) for extreme \(\hat p\) (e.g. \(\hat p = 0.02\) on \(n=50\)). For \(p \lesssim 5\%\) or \(p \gtrsim 95\%\) switch to Wilson or Clopper-Pearson: statsmodels.stats.proportion.proportion_confint(q, n, method="wilson").

ImportantTwo-sided vs one-sided CI

The CI should match the test — it uses the same Type I error budget:

  • two-sided → classic \([\hat p - z_{1-\alpha/2}\cdot SE,\ \hat p + z_{1-\alpha/2}\cdot SE]\). For \(\alpha = 0.05\) that’s \(z = 1.96\).
  • greater (H₁: \(\mu > \mu_0\)) → only a lower bound: \([\hat p - z_{1-\alpha}\cdot SE,\ 1]\). Reads as “\(\mu\) is at least \(L\) with confidence \(1-\alpha\)”.
  • less (H₁: \(\mu < \mu_0\)) → only an upper bound: \([0,\ \hat p + z_{1-\alpha}\cdot SE]\). Reads as “\(\mu\) is at most \(U\)”.

Why is this not “the same CI with one bound missing”? Because \(z_{1-\alpha} < z_{1-\alpha/2}\) (\(1.645 < 1.96\) at \(\alpha=0.05\)): a one-sided CI is tighter but says less — you trade away information about the other bound for a tighter first one.

NoteDeeper dive: Wilson and Clopper-Pearson CIs

The Wald interval relies on the normal approximation and plugs in the observed \(\hat p\) for \(SE\). That breaks it in three situations:

  • Small \(n\): the approximation itself is inaccurate.
  • Extreme \(\hat p\): \(SE = \sqrt{\hat p (1-\hat p)/n} \to 0\) as \(\hat p \to 0\) or \(\hat p \to 1\), so the CI is artificially narrow.
  • \(\hat p = 0\) or \(\hat p = 1\): the Wald CI collapses to a single point — “with 95% confidence the probability is exactly zero after 20 failures”.

There are two classical fixes.

5.0.1 Wilson (1927) — test inversion

Instead of fixing \(\hat p\) and computing \(SE\) from it, Wilson inverts the test itself: which values of \(p_0\) are consistent with the data at level \(\alpha\)?

\[\left|\hat p - p_0\right| \leq z_{1-\alpha/2}\sqrt{\dfrac{p_0(1-p_0)}{n}}.\]

Squaring gives a quadratic in \(p_0\). Its two roots are the bounds:

\[\text{CI}_{\text{Wilson}} = \dfrac{1}{1 + z^2/n}\left[\hat p + \dfrac{z^2}{2n} \pm z \sqrt{\dfrac{\hat p(1-\hat p)}{n} + \dfrac{z^2}{4n^2}}\right]\]

The center of the interval — \((\hat p + z^2/(2n))/(1 + z^2/n)\) — is shrunk towards \(0.5\) (acts as a weak Bayesian prior). The interval is always in \([0, 1]\).

5.0.2 Clopper-Pearson (1934) — exact

No approximations; the bounds come straight from the binomial distribution:

  • \(p_L\) — the smallest \(p\) for which \(P(X \geq q \mid p) \geq \alpha/2\).
  • \(p_U\) — the largest \(p\) for which \(P(X \leq q \mid p) \geq \alpha/2\).

Thanks to the Binomial↔︎Beta relationship these have a closed form via beta-distribution quantiles \(B_q\):

\[p_L = B_{\alpha/2}(q,\ n - q + 1),\qquad p_U = B_{1-\alpha/2}(q + 1,\ n - q).\]

Edge cases: \(p_L = 0\) when \(q = 0\) and \(p_U = 1\) when \(q = n\). Coverage is guaranteed \(\geq 1 - \alpha\) (the method is conservative).

from scipy.stats import beta as beta_dist
from statsmodels.stats.proportion import proportion_confint

def wald_ci(q, n, alpha=0.05):
    p_hat = q / n
    se    = np.sqrt(p_hat * (1 - p_hat) / n)
    z     = norm.ppf(1 - alpha/2)
    return (p_hat - z*se, p_hat + z*se)

def wilson_ci(q, n, alpha=0.05):
    p_hat = q / n
    z     = norm.ppf(1 - alpha/2)
    denom = 1 + z**2 / n
    center     = (p_hat + z**2 / (2*n)) / denom
    half_width = z / denom * np.sqrt(p_hat*(1-p_hat)/n + z**2/(4*n**2))
    return (center - half_width, center + half_width)

def clopper_pearson_ci(q, n, alpha=0.05):
    lower = 0.0 if q == 0 else beta_dist.ppf(alpha/2, q, n - q + 1)
    upper = 1.0 if q == n else beta_dist.ppf(1 - alpha/2, q + 1, n - q)
    return (lower, upper)

5.0.5 Coverage probability — graphical comparison

The most important property of a CI is how often it actually covers the true \(p\). For the binomial distribution this can be computed exactly (no simulation needed) by enumerating all \(n+1\) possible values of \(q\):

\[\text{Coverage}(p) = \sum_{q=0}^{n} \mathbb{1}\bigl[p \in \text{CI}(q)\bigr] \cdot \binom{n}{q} p^q (1-p)^{n-q}.\]

def exact_coverage(n, p_true, method, alpha=0.05):
    cov = 0.0
    for q in range(n + 1):
        lo, hi = method(q, n, alpha)
        if lo <= p_true <= hi:
            cov += binom.pmf(q, n, p_true)
    return cov

  • Wald — only when \(n \cdot \hat p \geq 30\) and \(n \cdot (1 - \hat p) \geq 30\), or as a back-of-envelope estimate. Wald is pedagogically valuable because its formula transparently shows the role of \(SE\) — but in production, take Wilson.

From one sample to two

my_z_test is the right tool when you have a fixed baseline — e.g. SLA monitoring (“is the production conversion \(\geq 10\%\) this week?”) or comparing this week against history. But a proper A/B test runs two cohorts in parallel — and both contain noise.

We add a sibling function my_two_sample_z_test. The old function does not go away — it stays useful for baseline-style scenarios: SLA monitoring, regression detection, sanity checks against history.

6 Version 5 — two-sample Z-test

Instead of a fixed \(\mu_0\) we use a pooled estimate:

\[ \hat{p}_{\text{pool}} = \dfrac{x_1 + x_2}{n_1 + n_2},\qquad SE_{\text{pool}} = \sqrt{\hat{p}_{\text{pool}}(1-\hat{p}_{\text{pool}}) \left(\tfrac{1}{n_1} + \tfrac{1}{n_2}\right)}. \]

def my_two_sample_z_test(x1, n1, x2, n2,
                         alternative="two-sided",
                         alpha=0.05, conf_int=False):
    """
    Two-sample Z-test for the difference in proportions.
    Convention: alternative is about p1 - p2 (control - test if you pass them in that order).
    """
    p1, p2 = x1 / n1, x2 / n2

    # Pooled SE — for the test
    p_pool  = (x1 + x2) / (n1 + n2)
    se_pool = np.sqrt(p_pool * (1 - p_pool) * (1 / n1 + 1 / n2))
    z       = (p1 - p2) / se_pool

    if alternative == "greater":
        p_value = 1 - norm.cdf(z)
    elif alternative == "less":
        p_value = norm.cdf(z)
    elif alternative == "two-sided":
        p_value = 2 * (1 - norm.cdf(abs(z)))
    else:
        raise ValueError(f"Unknown alternative: {alternative!r}")

    if not conf_int:
        return p_value

    # Unpooled SE — for the CI (no H0 assumption)
    se_unp = np.sqrt(p1 * (1 - p1) / n1 + p2 * (1 - p2) / n2)
    diff   = p1 - p2

    # CI matches alternative — same α budget.
    if alternative == "two-sided":
        z_crit  = norm.ppf(1 - alpha / 2)
        ci_low  = diff - z_crit * se_unp
        ci_high = diff + z_crit * se_unp
    elif alternative == "greater":
        # H1: p1 > p2 → one-sided lower bound on (p1 - p2)
        z_crit  = norm.ppf(1 - alpha)
        ci_low  = diff - z_crit * se_unp
        ci_high = 1.0
    else:  # less — H1: p1 < p2 → one-sided upper bound
        z_crit  = norm.ppf(1 - alpha)
        ci_low  = -1.0
        ci_high = diff + z_crit * se_unp

    return {
        "z_stat": z,
        "p_value": p_value,
        "estimate_diff": diff,
        "ci_low":  ci_low,
        "ci_high": ci_high,
    }

NotePower formula for the two-sample Z (one-sided)

\[ \text{Power}(\text{greater}) = 1 - \Phi\!\left( \dfrac{z_{1-\alpha} \cdot SE_{\text{null}} - (p_1 - p_2)}{SE_{\text{alt}}} \right), \] with \(SE_{\text{null}}\) computed at \(\bar p = (p_1+p_2)/2\) (pooled under \(H_0\)) and \(SE_{\text{alt}}\) at the actual \(p_1, p_2\).

def my_two_sample_z_power(p1, p2, n1, n2,
                          alpha=0.05, alternative="two-sided"):
    """
    Power of the two-sample Z-test on proportions.
    Convention: Z = (p1 - p2) / SE_pool; alternative is about p1 - p2.
    """
    delta   = p1 - p2
    p_bar   = (p1 + p2) / 2
    se_null = np.sqrt(p_bar * (1 - p_bar) * (1 / n1 + 1 / n2))
    se_alt  = np.sqrt(p1 * (1 - p1) / n1 + p2 * (1 - p2) / n2)

    if alternative == "greater":
        z_a = norm.ppf(1 - alpha)
        return 1 - norm.cdf((z_a * se_null - delta) / se_alt)
    if alternative == "less":
        z_a = norm.ppf(1 - alpha)
        return norm.cdf((-z_a * se_null - delta) / se_alt)
    if alternative == "two-sided":
        z_a   = norm.ppf(1 - alpha / 2)
        right = 1 - norm.cdf((z_a * se_null - delta) / se_alt)
        left  = norm.cdf((-z_a * se_null - delta) / se_alt)
        return right + left

    raise ValueError(f"Unknown alternative: {alternative!r}")

plan_ab_experiment via bisection — faster and more precise than a grid:

def plan_ab_experiment(p1, mde, alpha=0.05, target_power=0.8,
                       alternative="less", n_lo=10, n_hi=500_000):
    """
    How many users per group are needed to detect an effect of size `mde`?
    Bisection: ~log2(n_hi) steps instead of thousands.
    """
    # alternative gives the effect sign: "less" → p2 > p1; "greater" → p2 < p1
    sign = +1 if alternative != "greater" else -1
    p2   = p1 + sign * mde

    # Sanity: is n_hi enough?
    if my_two_sample_z_power(p1, p2, n_hi, n_hi, alpha, alternative) < target_power:
        return None

    while n_hi - n_lo > 1:
        n_mid = (n_lo + n_hi) // 2
        if my_two_sample_z_power(p1, p2, n_mid, n_mid, alpha, alternative) >= target_power:
            n_hi = n_mid
        else:
            n_lo = n_mid
    return n_hi

def my_ab_test(x1, n1, x2, n2,
               alternative="two-sided",
               alpha=0.05, target_power=0.8):
    """
    Full report for an A/B test. Reuses my_two_sample_z_test + my_two_sample_z_power.
    """
    # 1) Test + CI (reuse V5)
    test = my_two_sample_z_test(x1, n1, x2, n2,
                                alternative=alternative,
                                alpha=alpha, conf_int=True)

    p1, p2 = x1 / n1, x2 / n2
    lift   = p2 - p1

    # 2) Power at the observed data (reuse V6)
    power_obs = my_two_sample_z_power(p1, p2, n1, n2, alpha, alternative)

    # 3) MDE detectable at the current n_avg (bisection)
    n_avg = (n1 + n2) // 2
    sign  = +1 if alternative != "greater" else -1

    def has_power(mde):
        return my_two_sample_z_power(p1, p1 + sign * mde, n_avg, n_avg,
                                     alpha, alternative) >= target_power

    mde_at_n = None
    lo, hi = 0.0, min(p1, 1 - p1)
    if has_power(hi):
        for _ in range(40):
            mid = (lo + hi) / 2
            if has_power(mid):
                hi = mid
            else:
                lo = mid
        mde_at_n = hi

    decision = "REJECT H0" if test["p_value"] <= alpha else "do not reject H0"
    conf_pct = int((1 - alpha) * 100)

    # CI format matches alternative (same as V4/V5).
    if alternative == "two-sided":
        ci_line = (f"{conf_pct}% CI for lift = p2-p1: "
                   f"[{-test['ci_high']*100:+.2f}, {-test['ci_low']*100:+.2f}] pp")
    elif alternative == "less":   # H1: p1 < p2 → lift > -ci_high
        ci_line = (f"{conf_pct}% one-sided lower bound for lift: "
                   f"≥ {-test['ci_high']*100:+.2f} pp")
    else:                         # greater — H1: p1 > p2 → lift < -ci_low
        ci_line = (f"{conf_pct}% one-sided upper bound for lift: "
                   f"≤ {-test['ci_low']*100:+.2f} pp")

    summary_lines = [
        f"n1={n1}, x1={x1} (p1={p1:.4f})  vs  n2={n2}, x2={x2} (p2={p2:.4f})",
        f"lift = {lift:+.4f} ({lift*100:+.2f} pp)",
        f"Z = {test['z_stat']:+.3f},  p-value = {test['p_value']:.4f}  →  {decision}",
        ci_line,
        f"power at observed = {power_obs:.3f}",
    ]
    if mde_at_n is not None:
        summary_lines.append(
            f"MDE at n_avg={n_avg}, power={target_power}: {mde_at_n*100:.2f} pp"
        )

    return {
        "p1": p1, "p2": p2, "lift": lift,
        "z_stat": test["z_stat"],
        "p_value": test["p_value"],
        "reject": test["p_value"] <= alpha,
        "ci_diff": (test["ci_low"], test["ci_high"]),
        "power_at_observed": power_obs,
        "mde_at_n": mde_at_n,
        "summary": "\n".join(summary_lines),
    }

9 Sanity check: A/A simulation

Before trusting any A/B conclusions, stress-test the infrastructure with an A/A run — the same test applied to two groups with the same \(p\). If everything is wired correctly, the false-positive rate must be close to \(\alpha\).

def simulate_AA(p_shared, n_per_group, n_simulations=2000,
                alpha=0.05, alternative="two-sided"):
    """
    A/A simulation: estimates the empirical FPR.
    """
    rejections = 0
    for _ in range(n_simulations):
        s1 = binom.rvs(1, p_shared, size=n_per_group)
        s2 = binom.rvs(1, p_shared, size=n_per_group)
        p_val = my_two_sample_z_test(int(s1.sum()), n_per_group,
                                     int(s2.sum()), n_per_group,
                                     alternative=alternative)
        if p_val <= alpha:
            rejections += 1
    return rejections / n_simulations

11 Pitfalls of interpretation

WarningThree sentences a manager should never hear

❌ “\(p = 0.03\), so with probability 97% our button is better.” Conflates \(p\)-value with \(P(H_1 \mid \text{data})\).

❌ “The test showed no significant effect, so the button does not work.” Absence of evidence ≠ evidence of absence. Always ask about MDE: did the test even have a chance to detect the effect?

❌ “Let’s stop — we already see \(p < 0.05\) at 500 users.” Classic peeking: it inflates the FPR.

ImportantWhat the CI does not mean

❌ “There is a 95% probability that the true \(\mu\) lies in this interval.”

✅ “If we repeated the experiment many times, 95% of the constructed intervals would cover the true \(\mu\).”

The interval is random; the parameter is not.

WarningPooled vs unpooled SE — do not mix them up
  • Test of \(H_0: p_1 = p_2\): pooled SE (assume equal proportions).
  • CI for \(p_1 - p_2\): unpooled SE (describe the observed difference, no assumption).

A common bug in home-grown A/B calculators.

WarningMultiple testing

If you simultaneously test \(k\) variants at level \(\alpha=0.05\), the probability of at least one false positive grows as \(1 - (1-\alpha)^k\). For \(k=3\) that’s already \(\approx 14\%\). Simple fixes:

  • Bonferroni: use \(\alpha/k\) for each test (conservative).
  • Holm–Bonferroni or Benjamini–Hochberg (FDR control) — less conservative. Implementations: statsmodels.stats.multitest.multipletests.

Pre-launch self-check

If any of these is “no” — it is too early to launch.
